# DiscussionGroups 会话改动说明（2026-04-06）

本文档用于汇总本次长会话内最终落地的主要改动，方便后续回溯、测试与继续迭代。

## 1. 首页与全局配置

- 首页增加了时间入口，手动设置时间后不再是固定死时间，而是保存“设置时间与当前时间的差额”，后续 prompt 读取的是差额修正后的当前时间。
- 首页增加了规则入口，prompt 规则统一拆成框架化配置，并支持：
  - 框架顺序调整
  - 框架内条目拖拽排序
  - 内置文字可编辑
  - 自定义文字可新增、删除、排序
  - `{{userName}}` 这类运行时变量锁定不可误改
- JSON 导入导出流程中，导入默认改成全选。
- 首页左上角版本号与更新时间已纳入每次改动后的固定更新流程。

## 2. Chat 回复链路与渲染稳定性

- 修复了多种 chat 页消息丢失、返回后不显示、重进后只剩第一条、重回后旧消息未替换等问题。
- 回复按钮与“正在输入……”状态做了绑定，只有实际进入请求流程后才会显示对应状态。
- AI 回复支持续写：当用户没有新消息时，也可以主动要求 AI 继续往下说。
- 回复按钮在请求进行中会锁定，避免同一轮请求重复触发；请求结束或失败后恢复可点击。
- AI 多条回复的气泡显示已改为分条渐显，并对手机端的滚动/重绘问题做了多轮修复。
- 离开 chat 页时，后台仍可继续接收消息；当不在当前角色会话内时，顶部可收到消息提示。
- 修复了手机端输入框被快速收起、无法输入文字的问题。
- 修复了 chat 返回逻辑，1v1 对话页返回时会回到聊天列表页，而不是直接退回首页。

## 3. Chat 功能增强

- 工具栏补充了照片发送能力：
  - 通过拍照链路发送
  - 允许填写图片描述
  - 聊天中以图片卡片形式展示
  - AI 也可按约定格式主动发送图片消息
- 聊天气泡菜单补充“心声”功能：
  - 发送最近 10 轮上下文、角色设定、用户设定给 AI
  - 输出第一人称、单段、自然口语化的心理活动描述
- 支持引用回复、定位消息、图片消息等结构化消息类型，并纳入 prompt 感知。
- “重回”功能支持重新生成上一轮 AI 回复，并修复了首轮不触发、旧气泡残留、替换失败等问题。

## 4. Prompt 框架与 Chat Prompt 结构

- Chat 主 prompt 结构已从三段扩展为四段：
  - `context_library`
  - `persona_alignment`
  - `current_state_awareness`
  - `output_standard`
- `current_state_awareness` 新增为独立框架，用于承载：
  - 地点状态
  - 时间感知
  - 用户提及时段与对应日程
  - 当前日程感知
- 地点、时间、日程等“当前状态”信息不再混在 `context_library` 中。
- `presence_update` 的格式要求与输出约束已归入 `output_standard`，不再混在场景背景里。
- 续写提示不再塞在 `persona_alignment` 中，而是改为一条合成的用户消息追加到请求内容里，使模型更明确地把它理解成“主动续写请求”。

## 5. Chat 续写提示

- 新增独立可配置项：`Chat 续写提示`
- 这套配置可在规则页中直接查看和修改，当前包括：
  - 续写请求主句
  - 时间说明句
  - 4 档等待语气配置
    - 短间隔
    - 中间隔
    - 长间隔
    - 超长间隔
- 等待时间的影响方式已调整为：
  - 不直接把“等了多久”当正文说出来
  - 只体现在语气变化、犹豫感、重新开口方式里
- 自动二次补发 API 已取消，续写失败时不会再系统自发补打一轮请求。

## 6. 时间、地点与日程感知

- 当前日程注入 prompt 时，不只会写“正在做什么”，还会补充：
  - 已进行多久
  - 预计还会持续多久
- 当用户消息中明确出现“今天 / 明天 / 后天 / 昨天 / 下周 / 下周一”等时间词时，会主动匹配该时间对应的用户与角色日程，并作为补充信息注入 prompt。
- 日程注入 prompt 前会检查可见性，对角色不可见的日程不会进入 prompt。
- 1v1 聊天进入时，会根据用户/角色当前日程刷新地点与线上/线下状态。
- 日程支持常用地点，并可用于地点联动与状态刷新。

## 7. 跨日小时日程

- 小时类型日程已支持跨天，但限制为不超过 24 小时。
- 例如 `23:00 - 02:00` 会被视为持续到次日凌晨。
- 日程页展示、保存校验、冲突判断、聊天 prompt 感知都已同步支持跨日小时日程。

## 8. 论坛与其他 Prompt 修正

- 论坛 prompt 调整后，论坛页签信息不再放在普通背景信息中，其重要性高于世界书。
- 论坛回复主贴时，会确保论坛信息进入 prompt，不再漏掉论坛上下文。
- Chat 回复格式进一步约束：
  - 括号外遇到逗号或句号尽量换行
  - 括号内的行为描述不受这条规则影响
- 心声 prompt 调整为第一人称、单段、自然说话的描述风格。

## 9. 记忆与总结

- 总结触发阈值提升到 100，并支持每个角色在 1v1 对话设置中单独配置总结轮数。
- 总结日志支持编辑文字和重要度。
- 记忆注入继续区分：
  - 核心记忆：稳定影响态度与判断
  - 情景记忆：仅在自然触发时回想
- 当前 Chat prompt 中：
  - 核心记忆进入 `persona_alignment`
  - 情景记忆进入 `context_library`

## 10. 白名单 / 隐私覆盖

- 修复了白名单反解析失败、未配置命中后回填异常的问题。
- 修复了最近命中列表中“加入排除”后整页被清空的问题。
- 最近命中、白名单、排除词之间的联动更稳定，不会误清整组数据。
- 白名单手机端 UI 已按移动端重新压缩，减少冗余占位。
- 白名单单条词项在手机端保持“标签 + 删除”同一行显示。

## 11. 日志与调试

- API Log 页面不再省略长文，方便完整回溯 prompt 和返回内容。
- 对续写、重回、状态更新、消息回复等链路的日志记录做了多轮补强，便于定位“有返回但前端没显示”这一类问题。

## 12. 当前规则与测试关注点

后续如果继续测试，建议优先关注以下几项：

1. 主动续写时，模型是否仍会只返回 `presence_update` 而不给正文。
2. 手机端多条回复是否按节奏逐条出现，而不是首条后卡住再一口气弹出。
3. `Chat 续写提示` 配置修改后，模型语气是否确实跟随变化。
4. 跨日小时日程在日程页与 chat prompt 中是否保持一致。
